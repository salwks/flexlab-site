# FLEXLAB 사이트 리디자인 — 변경 사항 및 해결 기록

## 개요

기존 flexlab.genoray.com(아임웹 기반)을 Next.js 기반의 실험적 인터랙티브 사이트로 전면 재구축.

---

## 기존 사이트 분석 및 문제점

### 플랫폼
- **아임웹(imweb)** 템플릿 기반. 무거운 JS 로딩, 커스터마이징 한계.
- `robots: noindex` 설정으로 검색엔진 노출 차단 상태.

### 디자인
- 2010년대 기업 소개서 스타일. 원형 벤다이어그램, 컬러 카드 나열.
- 히어로가 YouTube 임베드 + 큰 로고 텍스트뿐 — 첫인상이 약함.
- 다크 톤(`#363636`) 위주로 무거운 느낌.

### 콘텐츠
- About: "FLEXLAB이 하는 일" 벤다이어그램 하나로 추상적 설명.
- R&D: Framework/Component/Tool/Engine 카테고리에 컬러 박스 나열. 각 기술이 무엇인지 구체적 설명 없음.
- Careers: 채용절차 → 인재상(원형 이미지) → 복리후생(텍스트 나열) → 사람인/잡코리아 링크. 사람인 스타일.
- Contact: 네이버 지도 + 주소. 이메일 하나만 있고 문의 양식 없음.
- 뉴스/블로그가 2023년에 멈춰 있어 업데이트 안 되는 인상.

### 기술적 문제
- URL이 `/33`, `/34`, `/35` 같은 의미 없는 숫자 ID.
- OG description이 모든 페이지에서 동일 — SEO 차별화 없음.
- 모바일에서 일부 레이아웃 깨짐.

## 새 사이트 — 해결 및 개선

### 1. 인터랙티브 로고 히어로

**문제**: 히어로가 YouTube 영상 하나뿐. 방문자에게 "소프트웨어 연구소"라는 정체성을 전달하지 못함.

**해결**: FLEXLAB 로고의 직선 구조를 활용한 Canvas 기반 인터랙티브 히어로.
- 로고의 각 선분을 물리 엔진으로 시뮬레이션.
- 마우스 호버 시 선분이 수직 방향으로 변형되고 스프링 물리로 진동 복귀.
- 양 끝점 고정 + 중앙부가 더 크게 변형되는 기타줄 물리.

### 2. 랜덤 히어로 엔진

**문제**: 매번 같은 첫 화면 → 재방문 동기 부족.

**해결**: 10개 히어로 변형 중 매 방문마다 랜덤 선택.
| 변형 | 기술 | 설명 |
|------|------|------|
| Wireframe | Canvas 2D + 물리 | 기타줄 튕김 |
| Inverted | Canvas 2D | 검정 배경 + 흰 로고 + 먼지 파티클 |
| Particle | Canvas 2D | 2,500개 점이 모여서 로고 형성, 마우스로 흩어짐 |
| Glitch | Canvas 2D | 디지털 글리치, 색 분리 슬라이스 |
| Ascii | Canvas 2D | 매트릭스 코드 스타일 문자 로고 |
| DotMatrix | Canvas 2D | LED 전광판 도트 그리드 |
| Ink | Canvas 2D | 수묵화 붓터치 드로잉 애니메이션 |
| TypoMotion | Framer Motion | 글자별 스프링 등장 + 호버 효과 |
| 3D | Three.js WebGL | 3D 박스 로고, 마우스 회전 |
| Shader | WebGL GLSL | 노이즈 디스토션 + 스캔라인, 시드 기반 무한 유니크 |

### 3. 프로시저럴 파라미터 생성

**문제**: 10개 변형이라도 결국 루프.

**해결**: 각 변형 내부에서 파라미터를 프로시저럴하게 생성.
- 10개 색상 팔레트 (시간대별 가중치: 밤=다크, 아침=따뜻한 톤).
- 선 굵기(2.5~5px), 로고 스케일(0.6~0.85), 물리 상수, 파티클 수 등 연속값 랜덤.
- 셰이더 시드로 매 프레임 유니크.
- **10개 변형 × 10개 팔레트 × 무한 연속값 = 사실상 무한 조합.**

### 4. 제너레이티브 사운드

**문제**: 기존 사이트에 사운드 없음.

**해결**: Tone.js 기반 Cubesato 스타일 앰비언트 엔진.
- C major pentatonic 스케일 — 어떤 조합이든 불협화음 없음.
- 패드(7초) + 벨(11초) + 베이스(13초) 소수 간격 루프 → 반복 주기 1,001초(≈16분).
- 로고 인터랙션 시 PluckSynth로 음 트리거 (선분 인덱스 → 스케일 음 매핑).
- 변형별 mood(bright/dark/ethereal)로 필터/리버브 자동 조절.
- 기본 뮤트 상태 + 우하단 토글 버튼. 브라우저 자동재생 정책 준수.

### 5. 콘텐츠 미니멀 리디자인

**문제**: 히어로는 실험적인데 스크롤하면 평범한 기업 템플릿. 톤 단절.

**해결**: 카드/그리드 완전 제거. 큰 타이포그래피 + 여백 + 스크롤 리빌.
- **About**: "FLEXLAB은 제노레이의 소프트웨어 연구소입니다." 한 문장이 큰 타이포로 존재감.
- **R&D**: 번호 매긴 기술 리스트 (01~08) + Products 인라인 텍스트. 카드 없음.
- **Careers**: 인재상 3줄 + 채용 프로세스 한 줄 + "지원하기 →" 링크 하나.
- **Contact**: 주소 자체가 큰 타이포 + 모노스페이스 연락처. 지도 제거.
- **Footer**: "FLEXLAB © 2026" 한 줄.
- 강조 텍스트에 그라디언트 애니메이션 적용 (8초 주기 색상 시프트).
- framer-motion `useInView`로 스크롤 시 요소 순차 등장.

## 해결한 기술적 이슈

### React Compiler 충돌
- **증상**: useEffect 의존성 배열 크기 변경 에러 반복.
- **원인**: Next.js 16의 React Compiler가 의존성을 자동 주입하면서 `interactive` prop 변경 시 배열 크기 불일치.
- **해결**: `next.config.ts`에서 `reactCompiler: false` 설정.

### Hydration Mismatch
- **증상**: SSR과 클라이언트에서 `Math.random()` 결과가 달라 배경색 불일치.
- **원인**: `useState` 초기화에서 랜덤 값 생성 → SSR/클라이언트 불일치.
- **해결**: 모든 랜덤화를 `useEffect` 안에서 수행 (클라이언트 전용) + `suppressHydrationWarning`.

### Tone.js triggerNote 에러
- **증상**: "Start time must be strictly greater than previous start time".
- **원인**: 여러 선분이 동시에 트리거되면서 같은 시간에 여러 노트 발생.
- **해결**: 50ms 쿨다운 + `Tone.now()` 명시적 전달.

### Canvas 로고 치우침/잘림
- **증상**: 일부 변형에서 로고가 오른쪽으로 치우치거나 B 글자 잘림.
- **원인**: 고정 `padX` 값 + 컨테이너 기반 캔버스 크기.
- **해결**: 전체 뷰포트(`window.innerWidth/Height`) 기반 캔버스 + 동적 중앙 정렬 `(cw - logoW) / 2`.

### 물리 엔진 렌즈 효과
- **증상**: 마우스 호버 시 모든 점이 방사형으로 밀려남 → 렌즈 왜곡처럼 보임.
- **원인**: 마우스 반발력이 방사형(radial)으로 적용됨.
- **해결**: 각 와이어의 수직 노말 벡터를 사전 계산하고, 힘을 수직 방향으로만 적용. 양 끝점 고정(pinned) + sin 엔벨로프로 중앙부만 변형.
